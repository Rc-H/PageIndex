You are given one JSON object describing a table-of-contents entry and several pages of a document. Your job is to determine the page number shown for that entry in the provided pages.

The provided pages may contain tags like <physical_index_X> and <physical_index_X> to indicate the physical location of the page X.

Important rules:
- Treat tags like <physical_index_X> only as annotations for physical PDF pages.
- Ignore page numbers printed in page headers, footers, or body text unless they are part of the table of contents entry itself.
- Printed page labels in headers or footers may be logical/document page numbers and may differ from physical PDF page indices.
- Your task here is to extract the page number shown in the table of contents entry itself, not to infer a physical page number from the surrounding pages.

The structure field is the numeric hierarchy index of the section in the table of contents. For example, the first section has structure index 1, the first subsection has structure index 1.1, the second subsection has structure index 1.2, etc.

If the section appears in the provided pages and has a page number shown in the table of contents, set "page" to that page number as an integer.

If the section does not appear in the provided pages or no page number is shown, set "page" to null.

Return exactly one JSON object in the following format:
{{
    "structure": "1.2",
    "title": "Section title",
    "page": 8
}}

Do not return a JSON array.
Do not wrap the object in another object.
Do not add explanations or an error field.
Keep "structure" and "title" unchanged from the given section object.
